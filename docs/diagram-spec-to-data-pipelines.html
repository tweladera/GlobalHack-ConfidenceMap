Challenge Area 1 of 3

Spec-to-Data Pipelines

Confidence Map · AI/works™ Hackathon 2026 · Team: Confidence Map

⚠️
El problema

Las especificaciones son texto no estructurado. Los equipos las interpretan, toman decisiones, y ese proceso es invisible, no trazable, y no comparable entre proyectos. Sin datos estructurados, la IA solo puede "hablar sobre" la spec — no razonar sobre ella.

⚙️
La solución

Cada agente usa Claude tool use con un JSON Schema estricto para transformar texto libre en hallazgos estructurados: nivel de confianza, score numérico, evidencia citada, supuestos explícitos, preguntas abiertas, y acción recomendada.

📊
El output

Datos Pydantic validados que alimentan el mapa visual, la tabla de decisiones, el Risk Heat Map, el backlog Jira-ready, el export PDF, y el chat de IA — todo desde el mismo pipeline de datos estructurados.

🔧 JSON Schema — report_findings tool
title string → título accionable, máx 100 chars
description string → explicación detallada del hallazgo
confidence enum "green" | "yellow" | "red"
confidence_score float → 0.0–1.0 · green:0.6–1.0 · yellow:0.3–0.75 · red:0.0–0.45
evidence string → cita textual exacta de la spec
assumptions string[] → supuestos explícitos del agente
needs_validation string[] → preguntas abiertas que el equipo debe validar
recommended_actionstring → siguiente paso concreto (1–2 frases)
category enum "ambiguity"|"contradiction"|"risk"|"gap"|"accessibility"|"cost"|"pattern"
