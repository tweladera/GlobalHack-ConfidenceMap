⚙️ Los 8 componentes del harness
🔀Orquestación en fases
Fase 1 (Spec Analyst, sequential) → Fase 2 (5 agentes, parallel) → Fase 3 (Consolidator, sequential). El output de cada fase informa a la siguiente.
🚦Semáforo de concurrencia
Semaphore(3) — máximo 3 agentes llamando a Claude simultáneamente. Evita saturar rate limits con 5 agentes paralelos.
🔁Retry + exponential backoff
MAX_RETRIES=2 · delays 2s → 4s. Cubre RateLimitError y errores 5xx del servidor.
✅Guardrails de calidad
_apply_guardrails(): score-level alignment, evidence placeholder, warning all-green. El harness valida el output antes de enviarlo al usuario.
🛡️Error isolation
call_agent() siempre retorna AgentResult, nunca lanza excepción. Los errores quedan en el resultado, el flujo continúa.
📡SSE Streaming
El usuario recibe eventos en tiempo real: AGENT_START, AGENT_COMPLETE, CONSOLIDATION_COMPLETE, ANALYSIS_COMPLETE.
📋Shared Blackboard
Findings del Spec Analyst formateados como XML y pasados a los 5 agentes de Fase 2. Comunicación via artefacto estructurado, no conversación directa.
🧪DEMO_MODE (test harness)
Con DEMO_MODE=true el harness usa mock results pre-generados con delays simulados. Flujo SSE idéntico al real. $0 de costo de API.
